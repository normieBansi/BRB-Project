Chapter 9: Deliverables and Future Scope
9.1 Introduction
This chapter outlines the concrete outcomes produced over the course of the BRB-NGFW project and establishes a roadmap for subsequent research and development. The project successfully bridged the gap between traditional open source firewalls and adaptive next generation security platforms. The deliverables represent a fully functional, reproducible proof of concept that fulfills all initial objectives. The future scope identifies specific architectural and algorithmic enhancements necessary to transition this academic framework into an enterprise ready deployment.
9.2 Project Deliverables
The successful execution of this project yielded a comprehensive suite of software components, configuration artifacts, and simulation tools. These deliverables collectively constitute the machine learning augmented sidecar architecture.
The primary deliverable is the machine learning detection engine, comprising the log parsing logic and the trained Isolation Forest pipeline. The parser efficiently normalizes heterogeneous telemetry from the OPNsense filterlog and Suricata intrusion detection systems into a unified feature space. The serialized scikit-learn pipeline encapsulates both the numerical and categorical preprocessing steps alongside the Isolation Forest algorithm, enabling rapid inference on incoming network flows without requiring continuous retraining.
The secondary deliverable is the FastAPI control service, which serves as the critical orchestration layer. This asynchronous web application successfully translates the abstract mathematical anomaly scores generated by the machine learning engine into actionable network security policies. By securely interfacing with the OPNsense REST API, the control service maintains the AUTO_BAN_IPS alias and executes automated firewall state reloads, thereby closing the response loop with minimal latency.
To ensure operational transparency, a complete observability dashboard was delivered using the Promtail, Loki, and Grafana technology stack. This centralized logging and visualization framework provides administrators with real time visibility into firewall event volumes, Suricata alert frequencies, algorithmic anomaly scoring, and the current enforcement state of the perimeter block lists.
To validate the system architecture, a repository of specialized Kali Linux attack simulation scripts was developed. These bash scripts leverage industry standard penetration testing tools such as hping3, Hydra, Nmap, and SQLMap to reliably reproduce a diverse spectrum of network layer and application layer threats. These scripts ensure that the experimental results remain strictly reproducible for future researchers.
Finally, the project delivered a complete, containerized laboratory architecture. Through declarative Docker Compose configurations and explicit virtual machine networking topologies, the entire sidecar integration can be rapidly deployed, tested, and modified by external developers without risking interference with their host networks.
9.3 Future Scope
While the current iteration of the BRB-NGFW system successfully demonstrates the viability of machine learning sidecars for open source firewalls, several technical avenues remain open for advanced research.
A primary area for future enhancement is the implementation of online learning models. The current Isolation Forest pipeline requires periodic offline retraining to adapt to shifting baseline traffic patterns. Transitioning to online algorithms, such as the Half-Space Trees anomaly detection framework, would allow the mathematical baseline to continuously evolve in real time, significantly reducing operational overhead and improving resilience against gradual concept drift.
The integration of deep learning architectures presents another realistic pathway for improving detection efficacy. While the current metadata feature engineering approach successfully identifies volumetric and behavioral anomalies, recurrent neural networks or Long Short-Term Memory models could be leveraged to analyze the sequential temporal patterns of individual packets within a specific TCP flow. This would provide superior accuracy in detecting slow, highly stealthy data exfiltration campaigns that evade basic volume thresholds.
The architecture could also be expanded to support federated threat intelligence sharing. In a distributed deployment, multiple independent BRB-NGFW instances could securely share their localized anomaly detection parameters and automated block lists with a centralized aggregation server. This federated approach would allow isolated open source firewall deployments to benefit from a collective, crowdsourced threat intelligence network similar to those maintained by commercial vendors.
Finally, future iterations should incorporate advanced Layer 7 payload analysis. While the current system intentionally focuses on metadata to maintain effectiveness against encrypted TLS traffic, integrating a specialized SSL inspection proxy would allow the machine learning models to extract deep application layer features. Analyzing HTTP headers, user agent strings, and structured API request payloads would dramatically increase the system's ability to detect sophisticated zero day web application exploits before they trigger traditional IDS signatures.
